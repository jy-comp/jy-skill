# mode=generate：AI 生成插件信息

## G1：生成名称和描述

基于 analyze 阶段的功能摘要，生成：

| 字段 | 要求 |
|------|------|
| **插件名称** | ≤30字，简洁有辨识度，体现核心功能 |
| **短描述** | ≤100字纯文本，一句话概括核心价值 |
| **详情描述** | 多段纯文本（CLI 会自动转富文本），包含功能介绍、使用方式、适用场景 |

**详情描述模板：**

```
功能介绍
[基于实际代码实现的核心功能说明，2-3 句]

使用方式
[基于实际点位类型说明在哪里可以看到/使用这个插件，1-2 句]

适用场景
[什么团队/项目/工作流适合使用，1-2 句]
```

## G2：获取分类列表并推荐

执行 CLI 获取分类列表：

```bash
npx @byted-meego/cli@builder list-categories
```

输出为 JSON 数组，每项包含 `id`、`name`（多语言 map）。

**AI 根据功能摘要推荐 1-3 个最匹配的分类。** 推荐依据：
- 点位类型（dashboard 类 → 数据分析，button/intercept 类 → 流程管理...）
- 代码中的核心逻辑（图表 → 数据可视化，审批 → 流程管理...）

> 若获取分类失败，跳过分类选择，后续可手动设置。
