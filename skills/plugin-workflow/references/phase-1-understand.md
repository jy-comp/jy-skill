# Phase 1：理解需求

## 1.1 捕获用户意图

用户可能的表达方式：
- "我需要一个在看板上展示燃尽图的功能"
- "帮我做一个按钮，点击后自动创建关联工作项"
- "我想在详情页加一个标签页显示测试报告"
- "做一个拦截器，提交时检查必填字段"

**不需要用户说"创建插件"**——AI 识别到需求需要自定义能力承载，自动进入插件开发流程。

## 1.2 推导点位类型

根据用户描述的功能，推导出需要的点位类型：

| 用户需求关键词 | 对应点位 | 说明 |
|--------------|---------|------|
| 看板、导航、页面 | `board` | 独立页面 |
| 视图、列表模式 | `view` | 工作项列表视图 |
| 标签页、详情页扩展 | `dashboard` | 工作项详情页标签 |
| 配置、设置 | `config` | 插件设置页 |
| 控件、字段 | `control` | 工作项表单控件 |
| 按钮、操作 | `button` | 工作项操作按钮 |
| 拦截、校验、阻止 | `intercept` | 事件拦截器 |
| 监听、通知 | `listen_event` | 事件监听 |
| 日程、计划 | `component` | 组件（schedule） |

一个功能可能需要多个点位配合，例如：
- "展示数据报表" → `dashboard`（展示）+ `config`（配置数据源）
- "批量操作工具" → `button`（触发）+ `board`（操作面板）

## 1.3 向用户确认

```
我理解你的需求是：

📋 功能：[一句话描述功能]
📌 实现方式：通过 Meego 插件，在 [位置] 添加 [能力]
🧩 需要的点位：
   - [点位类型1]：[用途说明]
   - [点位类型2]：[用途说明]（如有多个）

理解正确吗？如果需要调整可以告诉我。
```

**用户确认后进入 Phase 2。** 用户可以补充/修改/删除点位。

## 1.4 断点恢复

若用户中途回来继续，检查当前目录状态：
- 无 `plugin.config.json` → 从 Phase 2 开始
- 有 `plugin.config.json` 但无代码 → 检查点位是否已配置，从 Phase 2 或 3 开始
- 有代码 → 从 Phase 3 调试或 Phase 4 发布开始
