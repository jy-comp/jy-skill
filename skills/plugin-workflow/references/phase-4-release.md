# Phase 4：发布上线

完善插件信息并发布。用户只需最终确认。

## 4.1 完善信息 → plugin-polish

```
调用 /plugin-polish mode=pipeline
```

AI 基于已实现的代码和点位配置自动：
1. **分析**：读取点位配置 + 扫描代码，理解插件实际功能
2. **生成**：
   - 正式插件名称（替换工作名称）
   - 短描述（≤100字）
   - 详情描述（功能介绍 + 使用方式 + 适用场景）
3. **选择分类**：拉取分类列表，推荐最匹配的 1-3 个
4. **确认**：展示给用户确认，支持逐项修改
5. **提交**：调用 `update-description` 更新到后台

## 4.2 发布 → plugin-publish

```
调用 /plugin-publish mode=pipeline
```

**AI 自动处理，无需用户逐项输入：**
- **版本号**：不传，默认在上一版本基础上 patch +1
- **版本描述**：AI 基于当前代码改动自动总结生成

发布流程（串行）：
1. `update` — 同步配置
2. `release` — 构建 + 上传（获取 productVersion）
3. `publish` — 版本发布（输出分享链接）

## 4.3 完成

```
🎉 插件发布成功！

  插件名称：[正式名称]
  版本：[自动累加的版本号]
  分享链接：[URL]

用户可以通过分享链接安装使用此插件。
```
